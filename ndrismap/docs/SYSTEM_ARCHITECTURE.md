# System Architecture

## Technology Stack
- Frontend: HTML5, CSS3, Vanilla JavaScript
- Backend: PHP 8 (procedural)
- Database: MySQL / MariaDB
- Map: SVG + JavaScript (INTmap-based)
- Server: Apache (XAMPP)
- Version Control: Git & GitHub

## High-Level Flow
User Action → Frontend Form / Map Interaction  
→ JavaScript Validation  
→ PHP Controller  
→ SQL Database  
→ Response / Visualization

## Access Layers
- Public Layer: Map view, grievance submission
- Admin Layer: CRUD operations, analytics dashboard

## Design Constraints
- No frameworks
- Explainable logic for academic defense
- Modular PHP files
- Secure database interaction

## Why This Architecture
- Aligns with +2 PHP curriculum
- Scalable to real-world systems
- Minimal dependency risk
- Easy viva explanation
